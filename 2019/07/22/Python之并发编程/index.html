并发与并行

学过操作系统就应该听说过并发与并行的概念,两个概念是有所区别的,并发编程在同一个时刻不会多个操作进行,它是通过进程/任务之间的快速切换从而达到并发的效果.在python中的并发编程主要就是threading和asyncio.

并行则是在同一时刻运行多个任务,python中的multi-processing(多进程)就是并行编程

  • 并发通常应用于I/O操作频繁的场景
  • 并行一般用于CPU heavy的情况

什么是Futures

futures位于concurrent.futures和asyncio中,其会将处于等待状态的操作包裹起来放入队列,操作的状态可以随时查询.

并发编程之Asyncio

多线程编程存在一些缺点,例如多线程运行过程中容易被打断,出现race condition的情况.线程切换也存在一定的损耗,线程数不能无限增加,如果I/O操作较为heavy,则不能满足需求

什么是Asyncio

首先就是给出同步(Sync)和异步(Async)的概念

所谓同步就是操作是一个接着一个执行的,下一个操作必须等待上一个操作完成后才能执行

而异步则是不同操作之间可以相互交替执行,如果某个操作被阻塞,程序会找出可执行的操作

Asyncio工作原理

Asyncio其实也只要一个主线程,不过可以进行多个不同的任务,(特殊的future对象),不同的任务由event loop对象控制.

event_loop会维护两个任务列表[预备状态(随时准备运行)和等待状态(等待外部操作)],选取预备状态的任务运行,直至其把控制权交回.遍历等待状态列表,如果完成,则放入预备状态列表.对于Asyncio而言,其任务运行不会被外部因素打断.

并发编程的选择

一般而言

  • 如果是I/O较多并且I/O操作较慢时选择Asyncio(多任务/线程协同实行)
  • 如果I/O较多但是I/O操作快时可以选择多线程
  • 如果是占用cpu,使用多进程
