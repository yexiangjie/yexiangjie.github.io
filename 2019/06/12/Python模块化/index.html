Python模块化

发表于 2019-06-12 | 更新于: 2019-06-12 | | 阅读次数: |
字数统计: 691 | 阅读时长 ≈ 3

专栏的最后一讲基础篇主要讲述了Python的模块化,或许是很多项目都会使用到的技术吧.

简单模块化

简单的模块化就是把函数.类,常量划分到不同的文件,将他们放在同一个文件夹中,通过from your_file import function_name,class_name.来调用

项目模块化

在Linux系统中从根目录到叶子节点的路径称为绝对路径,例如/home/ubunt/Downloads/example.json

从一个文件到另一个文件,例如从同样的Downloads路径下的text.py到example.json就是可以通过/.

../…/downloads/example.json,这种路径称为相对路径

每个python文件都有自己的执行位置,可以通过sys.path.append修改路径,但是一般不这样做,

用pychram创建项目

通过from file_name来直接从根目录进行导入,而可以不是用…来返回上级路径,这是因为pycharm中正在使用import时会列表中寻找路径并增加到sys.path.

if name=’main’

import在导入文件时会将暴露在外面的代码都在执行一遍,如果你要把代码封装成模块,又可以执行时,就把要执行的代码放在main函数下面,

因为在在import时,name就改变为该模块的名字,不等于main,自然就不会执行了.

import

from model_name import * 会从模块中导入所有的函数和类,容易造成冲突,而import model_name也会导入所有的函数和类,但是使用的时候必须使用model.class_name
