python之所有方便快捷,就是因为有很多的好用的内置数据结构和第三方包,专栏分两期介绍了列表/元组/字典和集合,我在此总结一下

列表和元组

基础

列表和元组都是可以放置任意数据类型的有序集合

  • 列表是动态的，长度大小不固定，可以随意地增加、删除 元素

  • 而元组是静态的，长度大小固定，无法增加删减或者改变元素

特点

Python 中的列表和元组都支持负数索引，-1 表示最后一个数

另外，列表和元组都可以随意嵌套

常用的内置函数

  • count(item) 表示统计列表 / 元组中 item 出现的次数
  • index(item)表示返回回列表 / 元组中 item 第一次出现的索引
  • list.reverse() 和 list.sort() 分别表示翻转和排序,其中sort应该是有参数可以调的,来指明升序降序
  • reversd()和sorted()也是翻转和排序,但是会返回的新的列表/元组

列表和元组存储方式的差异

由于列表是动态的,所以其有指针指向对应的元素,以及其可以扩容的特性都增加了列表的存储空间,

想对而言列表应用于可变的元素,性能较元组略差

字典和集合

基础

字典是一系列无序元素的集合,大小可变,元素可以任意地删除和改变,每个元素是一对key和value的配对,其性能，字典的性能更优，特别是对于查找、添加和删除操作，字典都能在常数时间复杂度内完成.

集合是一系列无序的、唯一的元素组合。

字典访问可以直接索引键(key)，如果不存在，就会抛出异常

也可以采用get(key,”null”)的方式进行索引

索引

集合并不支持索引操作，因为集合本质上是一个哈希表

想要判断一个元素在不在字典或集合内，我们可以用 value in dict/set

字典和集合的工作原理

字典和集合本质上都是哈希表

插入操作
每次向字典或集合插入一个元素时，Python 会首先计算键的哈希值(hash(key)) ，再和mask= PyDicMinSize- 1做与操作，计算这个元素应该插入哈希表的位置index = hash(key) &mask。如果哈希表中此位置是空的,那么这个元素就会被插入其中。

而如果此位置已被占用，Python 便会比较两个元素的哈希值和键是否相等。

  • 若两者都相等，则表明这个元素已经存在，如果值不同，则更新值。
  • 若两者中有一个不相等，这种情况我们通常称为哈希冲突(hash collision) ,意思是两个元素的键不相等，但是哈希值相等。这种情况下，Python 便会继续寻找表中空余的位置，直到找到位置为止。

值得一提的是，通常来说，遇到这种情况，最简单的方式是线性寻找，即从这个位置开始，挨个往后寻找空位。当然，Python 内部对此进行了优化(这一点无需深入了解,你有兴趣可以查看源码，我就不再赘述) , 让这个步骤更加高效。

查找操作
和前面的插入操作类似，Python 会根据哈希值，找到其应该处于的位置;然后，比较哈希表这个位置中玩素的哈希值和键，与需要查找的元素是否相等。如果相等，则直接返回;如果不等,则继续查找，直到找到空位或者抛出异常为止。

删除操作
对于删除操作，Python 会暂时对这个位置的元素，赋于一个特殊的值,等到重新调整哈希表的大小时，再将其删除。
